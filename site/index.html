Open-source evidence gates for AutoResearch

Find out whether the idea failed, the run failed, or the claim is too early.

SciTriage is a plugin-style diagnostic layer for AutoResearch projects. It reads logs, patches, claims, and seed results, then tells the agent what to fix or verify next.

What question does it answer?

AutoResearch agents already know how to propose and run. SciTriage focuses on the uncomfortable part after the run.

Why did it fail?

Separate idea failure from implementation bugs, evaluator drift, unsupported claims, noisy metrics, and resource mismatch.

Should we continue?

Compare the one-shot delta to measured seed noise and promote only the strongest landmarks in each research direction.

Can we claim it?

Allow claims only when seed-group evidence clears the uncertainty margin; otherwise write down the missing evidence.

From harness to evidence

01

Read artifacts

Use the harness logs, changed files, candidate claims, and seed groups. No custom research agent required.

02

Choose a probe

Recommend the cheapest useful check: resource fit, more seeds, frozen evaluator, ablation, or claim rewrite.

03

Gate the claim

Return machine-readable JSON so an AutoResearch loop can continue, pause, or rewrite the research record.
